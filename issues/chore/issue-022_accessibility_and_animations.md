# Issue-022: アクセシビリティ・アニメーション・レスポンシブ対応

## 概要
MVPの全機能実装後に、アクセシビリティ・アニメーション・レスポンシブ対応を横断的に確認・整備する。
設計書のアクセシビリティ要件とアニメーション仕様をすべての画面・コンポーネントで満たす。

## タスク
- [ ] フォームのアクセシビリティを確認する
  - 各入力フィールドに `<label>` が設定されていること
  - エラーメッセージに `aria-live="polite"` が設定されていること
  - タブキーでの移動順序が適切であること
- [ ] 削除確認ダイアログのアクセシビリティを確認する
  - `role="alertdialog"` が設定されていること
  - ダイアログ表示時にフォーカスが移動すること
  - Escキーでキャンセルできること
- [ ] タッチターゲットサイズを確認する（すべてのボタンが44px × 44px以上）
- [ ] アニメーションを確認・実装する
  - ボタンタップ: 背景色を暗く（0.2秒）
  - 前回比表示: フェードイン（0.3秒）
  - トースト通知: スライドイン（0.3秒）
  - 画面遷移: スライドイン右から左（0.3秒）
  - カード削除: フェードアウト + 高さ縮小（0.3秒）
- [ ] レスポンシブ対応を確認する（PC時に最大幅600px、中央寄せ）
- [ ] コントラスト比を確認する（WCAG AA準拠）

## 完了条件
- スクリーンリーダーで主要操作が可能であること（手動確認）
- 全アニメーションが設計書の仕様通りに動作する
- PC・モバイルで適切なレイアウトが表示される
