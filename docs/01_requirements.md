# 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
過去問記録アプリ

### 1.2 バージョン
1.0.0

### 1.3 作成日
2026年2月14日

### 1.4 最終更新日
2026年4月25日

### 1.5 概要
応用情報技術者試験の午前試験における過去問演習の記録を管理するWebアプリケーション。日々の演習結果を記録し、正答率の推移を可視化することで、学習の進捗を客観的に把握し、試験に向けたモチベーションを維持・向上させる。

## 2. 目的・背景

### 2.1 開発目的
応用情報技術者試験の午前試験対策として、過去問演習の記録を簡単に管理できるツールを提供する。記録のハードルを下げ、継続的な学習習慣の形成を支援する。また、正答率の変化を可視化することで、自身の成長を実感し、試験に対するモチベーションを向上させる。

### 2.2 背景
資格試験の対策では過去問の反復演習が効果的だが、以下の課題がある:
- 演習結果を記録・管理するツールが少ない
- 自分の成長を客観的に把握しにくい
- 記録作業が面倒で継続しにくい

これらの課題を解決し、効率的な学習をサポートするアプリケーションを開発する。

## 3. 対象ユーザー

### 3.1 ターゲットユーザー
- 2026年、応用情報技術者試験を受験する人(開発者自身)
- 将来的には同試験を受験する他のユーザーも想定

### 3.2 ユーザーの課題・ニーズ
- 午前試験の過去問演習結果を簡単に記録したい
- 自分の学力の変化を数値で把握したい
- 成長を実感してモチベーションを維持したい
- 記録作業に時間をかけたくない(素早く入力したい)

## 4. 機能要件

### 4.1 MVP機能(必須)

#### 4.1.1 受験日設定機能
- **概要**: 試験日を設定し、残り日数を表示する
- **詳細**:
  - 受験予定日を入力できる
  - メイン画面に「試験まであと○○日」と表示
  - 設定した日付は変更可能
- **優先度**: 高

#### 4.1.2 過去問記録入力機能
- **概要**: 演習セッションの結果を素早く記録する
- **詳細**:
  - メイン画面に入力フォームを配置
  - 入力項目:
    - 実施日(デフォルト: 今日の日付)
    - 解いた問題数
    - 正答した問題数
  - 正答率を自動計算して表示
  - 記録ボタンを押すとデータを保存
- **優先度**: 高
- **備考**: 入力のハードルを最小限にするため、必要最小限の項目のみ

#### 4.1.3 前回との差分表示機能
- **概要**: 前回の記録と比較して正答率の変化を表示
- **詳細**:
  - 記録入力後、前回の正答率との差分を表示
  - 表示形式: 「前回比 +5%」「前回比 -3%」など
  - 前回の記録がない場合は差分を表示しない
- **優先度**: 高

#### 4.1.4 記録閲覧機能
- **概要**: 過去の記録一覧を表示する
- **詳細**:
  - メイン画面から遷移するボタンを配置
  - 記録一覧ページに以下を表示:
    - 実施日
    - 解いた問題数
    - 正答数
    - 正答率
    - 前回比
  - 新しい順に並べる
- **優先度**: 高

### 4.2 将来的な拡張機能(オプション)

#### 4.2.1 ユーザー認証機能
- **概要**: 複数ユーザーが利用できるようにする
- **詳細**:
  - ユーザー登録・ログイン機能
  - ユーザーごとにデータを分離
- **優先度**: 中

#### 4.2.2 カレンダー表示機能
- **概要**: 演習履歴をカレンダー形式で表示
- **詳細**:
  - 月間カレンダーに演習日をマーク
  - 日付をクリックするとその日の詳細を表示
  - 演習頻度を視覚的に把握できる
- **優先度**: 中

#### 4.2.3 正答率推移グラフ
- **概要**: 正答率の変化をグラフで可視化
- **詳細**:
  - 折れ線グラフで正答率の推移を表示
  - 期間指定機能(直近1週間、1ヶ月など)
  - 成長の傾向を視覚的に把握できる
- **優先度**: 中

#### 4.2.4 単元別記録機能
- **概要**: 単元ごとに記録・分析できるようにする
- **詳細**:
  - 記録時に単元を選択(テクノロジ系、マネジメント系、ストラテジ系)
  - 単元ごとの正答率を表示
  - 単元ごとの前回比を表示
- **優先度**: 中

#### 4.2.5 PC対応(レスポンシブデザイン)
- **概要**: PC画面でも快適に利用できるようにする
- **詳細**:
  - 大画面レイアウトの最適化
  - タブレット・デスクトップでの表示改善
- **優先度**: 低

#### 4.2.6 既存過去問アプリ連携
- **概要**: 他の過去問アプリのデータをインポート
- **詳細**: 検討中
- **優先度**: 低

#### 4.2.7 他の試験への対応
- **概要**: 資格試験を自ら設定し、応用情報技術者試験以外でも使えるようにする。
- **詳細**: 検討中
- **優先度**: 低

## 5. 画面設計(概要)

### 5.1 メイン画面(ホーム)
- 受験日までのカウントダウン表示
- 過去問記録入力フォーム
  - 実施日(日付ピッカー)
  - 解いた問題数(数値入力)
  - 正答した問題数(数値入力)
  - 記録ボタン
  - 正答率(自動計算・表示のみ)
- 前回比の表示エリア
- 「記録を見る」ボタン(記録一覧画面へ遷移)

### 5.2 記録一覧画面
- 過去の記録を一覧で表示
- 各記録に削除ボタン
- ホームに戻るボタン

## 6. データ設計(概要)

### 6.1 記録データ
各演習セッションごとに以下のデータを保存:
- ID(自動生成)
- 実施日(日付)
- 解いた問題数(整数)
- 正答数(整数)
- 正答率(小数点、自動計算)
- 作成日時(タイムスタンプ)

### 6.2 設定データ
- 受験予定日(日付)

## 7. 技術要件

### 7.1 プラットフォーム
- Webアプリケーション
- モバイルファースト設計(スマートフォン最適化)

### 7.2 技術スタック
- フロントエンド: React / TypeScript
- データ保存: LocalStorage
- ホスティング: Vercel / Netlify など(無料枠を利用)

### 7.3 非機能要件
- データの永続化(LocalStorageを使用、ブラウザのデータ削除に注意)
- シンプルで直感的なUI
- 高速な動作(記録入力が素早く完了する)
- モバイル環境での快適な操作性

## 10. 備考・補足

### 10.1 設計方針
- **シンプルさ優先**: 機能を詰め込まず、必要最小限から始める
- **継続性重視**: 記録のハードルを下げ、毎日使いたくなるUIを目指す
- **段階的な拡張**: MVPの完成を最優先し、その後機能を追加

### 10.2 今後の検討事項
- データのエクスポート/インポート機能(LocalStorageのバックアップ対策)
- クラウド同期機能(Firebase等への移行)
- 統計情報の表示(平均正答率、総演習回数など)
- 目標設定機能(目標正答率の設定など)
