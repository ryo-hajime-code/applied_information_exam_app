```
┌──────────────────────────────┐

│   │ 2026-02-14        [📅] │ │  ← 日付入力
│   └────────────────────────┘ │
│                              │
│   解いた問題数               │
│   ┌────────────────────────┐ │

│   └────────────────────────┘ │
│                              │
│   正答した問題数             │
│   ┌────────────────────────┐ │
│   │ (例: 60)               │ │  ← 数値入力(placeholder)

│                              │
│   ┌────────────────────────┐ │
│   │   📋 記録を見る         │ │  ← 遷移ボタン
│   └────────────────────────┘ │
│                              │
└──────────────────────────────┘
```

### 3.2 構成要素

#### A. カウントダウンエリア（受験日設定を兼ねる）

**受験日が設定済みの場合**:
```
┌──────────────────────────────┐
│  📅 試験まであと 65日    [✏️] │  ← 編集アイコンをタップで編集
└──────────────────────────────┘
```

**受験日が未設定の場合**:
```
┌──────────────────────────────┐
│  📅 受験日を設定する →        │  ← タップで入力欄を表示
└──────────────────────────────┘
```

**受験日設定UI（インライン展開）**:

カウントダウンエリアをタップ（または編集アイコンをタップ）すると、同じ場所にインラインで日付入力欄が展開される:

```
┌──────────────────────────────┐
│  📅 受験予定日               │
│  ┌────────────────────────┐ │
│  │ 2026-04-20        [📅] │ │  ← 日付ピッカー
│  └────────────────────────┘ │
│  ┌──────┐  ┌──────────┐    │
│  │ 保存  │  │ キャンセル │    │
│  └──────┘  └──────────┘    │
└──────────────────────────────┘
```

**設計判断**: 別ページ（/settings）ではなくインライン展開にした理由は、画面遷移を減らし、操作のハードルを下げるため。将来的に設定項目が増えた場合は独立した設定画面を検討する。

#### B. 記録入力フォーム

**フィールド一覧**:

| フィールド | 入力タイプ | デフォルト値 | placeholder | 必須 |
|-----------|-----------|-------------|-------------|------|
| 実施日 | date | 今日 | - | ✅ |
| 解いた問題数 | number | 空(未入力) | 例: 80 | ✅ |
| 正答した問題数 | number | 空(未入力) | 例: 60 | ✅ |

**バリデーション**:
- 解いた問題数: 1以上200以下の整数
- 正答した問題数: 0以上、解いた問題数以下の整数
- エラーはフィールド直下にインライン表示（赤文字）
- 該当フィールドを赤枠でハイライト

**正答率の表示**:
- 入力中にリアルタイムで計算・表示
- 小数点第1位まで表示
- 両方の数値が入力されるまで「---%」と表示

#### C. 前回比表示エリア

```
前回比: +8.0% ⬆   → 緑色(#34C759)
前回比: -3.5% ⬇   → 赤色(#FF3B30)
前回比: ±0.0% →   → グレー(#8E8E93)
```

**表示タイミング**: 記録ボタン押下後にフェードインで表示
**前回の定義**: 日付が最新の記録（同日複数の場合はcreatedAtで判定）

#### D. 遷移ボタン
- タップで記録一覧画面へ遷移

### 3.3 インタラクション

#### 記録追加フロー
```
1. ユーザーがフォームに入力
   ↓
2. 正答率がリアルタイムで計算・表示
   ↓
3. 「記録する」ボタンをタップ
   ↓
4. バリデーション実行
   ↓ (成功)
5. 保存前に前回の記録を取得（前回比計算のため）
   ↓
6. LocalStorageに保存
   ↓
7. 前回比を計算・表示
   ↓
8. フォームをクリア（日付は今日のまま、数値は空に）
   ↓
9. トースト通知「記録を追加しました ✓」（2秒で自動消去）
```

#### エラー時の表示

```
解いた問題数
┌────────────────────────────┐  ← 赤枠
│ -5                         │
└────────────────────────────┘
⚠️ 1以上の数値を入力してください
```

- エラーメッセージはフィールド直下にインライン表示
- 赤枠(#FF3B30) + 赤文字
- リアルタイムバリデーション（入力変更時にエラーを更新）

### 4.2 構成要素

#### A. ヘッダー
- 左: 戻るボタン(HOME画面へ)
- 中央: タイトル「記録一覧」

#### B. サマリー情報
```
📈 平均正答率: 81.3%
📝 総演習回数: 15回
```

**備考**: 要件定義書のMVP機能には明記されていないが、計算コストが低く有用なため含める。

#### C. 記録カード(1件)
```
┌──────────────────────────────┐
│  2026-02-16             [🗑] │  ← 日付 + 削除ボタン
│  80問中 68問正解              │
│  正答率: 85.0%                │
│  前回比: +1.0% ⬆              │
└──────────────────────────────┘
```

**表示ルール**:
- 日付降順（同日はcreatedAt降順）
- 最も古い記録は前回比なし（「--」と表示）
- 前回比の計算方法は03_database.md セクション5を参照

#### D. 削除機能

**削除ボタン**: 各カードの右上にゴミ箱アイコン(🗑)

**削除確認ダイアログ**:
```
┌──────────────────────────────┐
│                              │
│   この記録を削除しますか？    │
│                              │
│   2026-02-16                 │
│   80問中 68問正解             │
│   正答率: 85.0%               │
│                              │
│  ┌──────────┐ ┌──────────┐  │
│  │ キャンセル │ │   削除    │  │
│  └──────────┘ └──────────┘  │
│                              │
└──────────────────────────────┘
```

- 「削除」ボタンは赤色(#FF3B30)
- 削除後はカードがフェードアウトし、サマリー情報も更新される
- 削除後にトースト通知「記録を削除しました」
- 削除によって隣接レコードの前回比が変わる場合は再計算される

## 5. 共通UI要素

### 5.1 ボタンスタイル

| 種類 | 背景色 | 文字色 | 用途 |
|------|--------|--------|------|
| プライマリ | #007AFF | #FFFFFF | 記録する、保存 |
| セカンダリ | #F2F2F7 | #000000 | 記録を見る、戻る、キャンセル |
| デンジャー | #FF3B30 | #FFFFFF | 削除 |

共通: パディング上下16px・左右24px、角丸8px

### 5.2 入力フィールド
- 背景: #FFFFFF
- ボーダー: #D1D1D6（通常）、#007AFF（フォーカス）、#FF3B30（エラー）
- パディング: 12px、角丸: 8px

### 5.3 カード
- 背景: #FFFFFF
- 影: 軽いドロップシャドウ
- パディング: 16px、角丸: 12px、マージン: 上下8px

### 5.4 色使い

| 用途 | カラーコード |
|------|-------------|
| プライマリ | #007AFF |
| 成功(向上) | #34C759 |
| 警告(低下) | #FF3B30 |
| 中立 | #8E8E93 |
| 背景 | #F2F2F7 |
| 白 | #FFFFFF |
| テキスト | #000000 |

### 5.5 トースト通知
- 画面下部からスライドイン、2秒で自動消去
- 成功: 背景 #34C759
- 削除: 背景 #8E8E93

## 8. アクセシビリティ

### 8.1 フォーム
- 各入力フィールドに`<label>`を設定
- エラーメッセージを`aria-live="polite"`で通知
- タブキーでの移動順序を適切に設定
- 削除確認ダイアログに`role="alertdialog"`を設定

### 8.2 色覚対応
- 色だけに頼らない情報伝達（矢印アイコン併用）
- 十分なコントラスト比(WCAG AA準拠)

### 8.3 タッチターゲット
- ボタン最小サイズ: 44px × 44px
- ボタン間のスペース: 8px以上
